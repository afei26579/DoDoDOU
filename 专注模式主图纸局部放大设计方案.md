# 专注模式主图纸局部放大设计方案

## 目标

在不改变现有预览图效果的前提下，重构专注模式页面中的“主图纸区域”展示方式，使其从“整图显示”变为“当前正在拼的颜色的局部放大视图”，帮助用户更快定位正在拼的区域。

## 设计原则

1. **预览图保持不变**
   - 继续使用现有整图 canvas 渲染逻辑。
   - 继续作为“全局定位参考图”使用。
   - 不改变其绘制规则、缩放策略和交互方式。

2. **主图纸改为局部放大视图**
   - 只聚焦当前正在拼的颜色块。
   - 通过局部窗口放大，让用户更容易观察当前色块周围的上下文。
   - 主图纸区域不再承担整图预览职责。

3. **数据层统一，渲染层分离**
   - 两个区域共享同一份图纸数据 `patternResult`。
   - 预览图和主图纸使用不同的渲染函数。
   - 避免修改主图纸逻辑时影响预览图表现。

---

## 页面职责拆分

### 1. 预览图

**职责**：显示整张图纸中当前颜色的大概位置。

**要求**：
- 保持现有显示效果不变。
- 继续使用当前 canvas 渲染逻辑。
- 继续显示整图、完成状态、颜色高亮等全局信息。

### 2. 主图纸区域

**职责**：显示当前正在拼的颜色的局部放大区域。

**要求**：
- 根据当前色块计算局部窗口。
- 使用 `N * N` 的大格子进行展示。
- `N` 最小为 `10`。
- 根据当前色块所在图纸区域决定标尺位置。
- 只强调当前颜色，方便用户快速定位。

---

## 总体实现思路

推荐采用 **“主图纸新增局部放大 canvas 渲染器”** 的方案。

即：

- 现有预览图继续使用 `drawPatternPreview`，不做改动。
- 新增一个主图纸局部放大渲染函数，例如：`drawPatternFocusZoomView`。
- 专注模式页面根据当前颜色块，计算局部窗口参数后调用该渲染函数。

这样可以确保：

- 预览图逻辑稳定。
- 主图纸视觉效果独立可控。
- 后续可继续扩展标尺、放大倍数、辅助线等能力。

---

## 数据来源

专注模式页面已经具备以下核心数据：

- `patternResult`：完整图纸数据
- `currentColorEntry`：当前正在拼的颜色
- `currentBlock`：当前颜色对应的色块/块级信息
- `activeBlockCellKeys`：当前激活格子集合
- `completedCellKeys`：已完成格子集合

其中，主图纸局部放大视图主要依赖：

- `patternResult`
- `currentBlock.cells`
- `activeBlockCellKeys`

## 主图纸局部放大视图设计

### 1. 局部窗口大小 `N`

主图纸展示区域将不再渲染整张图，而是渲染一个局部窗口。当前产品实现中，局部窗口固定为 `10 × 10` 个格子。

#### 当前实现规则

- 可视窗口固定为 `10 × 10`。
- `N` 当前固定为 `10`，不随当前色块跨度自动变大。
- 当当前色块跨度小于或等于 `10 × 10` 时，主图纸窗口可以完整容纳当前色块。
- 当当前色块跨度超过 `10 × 10` 时，仍然保持 `10 × 10` 可视窗口，通过受限滑动和横纵分页移动查看超出部分。

#### 曾考虑的自适应方案

早期方案曾考虑使用以下方式自动扩大窗口：

```ts
N = Math.max(10, max(spanX, spanY) + padding * 2)
```

其中：

- `spanX = maxX - minX + 1`
- `spanY = maxY - minY + 1`
- `padding` 建议为 `2 ~ 4`

但当前实现已调整为：**固定 `10 × 10` 窗口 + 受限滑动/分页移动**。

#### 说明

- 固定 `10 × 10` 可以保证主图纸区域内单格大小稳定。
- 大色块不会压缩格子，而是通过移动窗口查看未显示的部分。
- 该策略更适合手机小屏下保持格子可读性。

---

### 2. 局部窗口定位

局部窗口需要围绕当前色块进行布局。实际显示窗口固定为 `10 × 10` 个格子，当当前色块跨度超过 `10 × 10` 时，通过受限滑动查看超出部分。

#### 步骤

1. 计算当前色块边界：
   - `minX`
   - `maxX`
   - `minY`
   - `maxY`

2. 计算当前色块跨度：
   - `spanX = maxX - minX + 1`
   - `spanY = maxY - minY + 1`

3. 根据“拼豆设置”中的操作习惯决定默认窗口锚点：
   - **左手模式**：优先展示当前色块右上角附近的 `10 × 10`
   - **右手模式**：优先展示当前色块左上角附近的 `10 × 10`

4. 当色块跨度不超过 `10 × 10` 时：
   - 当前色块可以完整显示在主图纸区域中。
   - 不需要开放对应方向的滑动。

5. 当色块跨度超过 `10 × 10` 时：
   - 主图纸仍然只显示 `10 × 10` 可视窗口。
   - 滑动范围由当前色块的横纵跨度决定。
   - 例如当前色块跨度为 `12 × 15`，则横向最多只需要滑动 `12 - 10 = 2` 格，纵向最多只需要滑动 `15 - 10 = 5` 格。

#### 受限滑动规则

- 滑动范围只覆盖当前色块需要查看的范围，不允许在整张图纸中无限拖动。
- **左手模式**：默认窗口贴近色块右上角，因此只允许向左、向下查看当前色块未显示的部分。
- **右手模式**：默认窗口贴近色块左上角，因此只允许向右、向下查看当前色块未显示的部分。
- 如果 `spanX <= 10`，横向滑动范围锁定为初始窗口位置。
- 如果 `spanY <= 10`，纵向滑动范围锁定为初始窗口位置。

#### 边界处理

- 如果窗口左侧超出图纸，窗口整体右移。
- 如果窗口右侧超出图纸，窗口整体左移。
- 如果窗口上侧超出图纸，窗口整体下移。
- 如果窗口下侧超出图纸，窗口整体上移。
- 受限滑动边界仍需被整张图纸边界裁剪，确保局部窗口始终合法。

这样可以保证：

- 局部窗口始终合法。
- 默认展示内容符合当前操作习惯。
- 小色块完整显示且不产生无意义拖动。
- 大色块只能在当前色块所需范围内受限滑动。

---

## 标尺显示规则

标尺位置由“拼豆设置”中的操作习惯决定，与局部窗口默认锚点保持一致。

### 标尺位置规则

| 操作习惯 | 横向标尺 | 纵向标尺 | 默认窗口锚点 |
| --- | --- | --- | --- |
| 左手模式 | 上侧 | 右侧 | 当前色块右上角 |
| 右手模式 | 上侧 | 左侧 | 当前色块左上角 |

### 标尺作用

标尺不仅是视觉装饰，还应提供明确的定位信息：

- 横向标尺：显示局部窗口内的列坐标。
- 纵向标尺：显示局部窗口内的行坐标。
- 支持每隔若干格显示数字，其余位置显示短刻度。

---

## 主图纸渲染内容

主图纸局部放大视图建议只突出以下内容：

1. **当前正在拼的颜色格子**
   - 作为当前视觉焦点。
   - 高亮边框或更强对比色处理。

2. **当前色块周边上下文**
   - 让用户知道当前颜色块和周围图案的位置关系。
   - 保持与背景格子的区分。

3. **标尺**
   - 显示在上/下/左/右对应边。
   - 帮助用户进行精确定位。

4. **必要的格线**
   - 维持大格子结构感。
   - 让局部区域更像“放大后的图纸”。

### 不建议保留的内容

为了避免干扰定位，主图纸局部视图中不建议强化以下内容：

- 整图缩略信息。
- 与当前颜色无关的过多装饰。
- 与预览图重复的全局表达。

---

## 渲染层设计建议

### 1. 预览图渲染器保持原样

继续使用现有的 `drawPatternPreview` 逻辑。

它负责：

- 整图展示。
- 当前颜色高亮。
- 已完成格子覆盖。
- 分割线显示。

### 2. 新增主图纸局部放大渲染器

建议新增一个独立函数，例如：

```ts
drawPatternFocusZoomView(params)
```

建议入参包含：

- `canvas`
- `pattern`
- `currentBlockCells`
- `windowBounds`
- `rulerPosition`
- `activeColorKey`
- `completedCellKeys`

#### 主要职责

- 计算局部格子绘制位置。
- 绘制 `N * N` 放大网格。
- 绘制标尺。
- 高亮当前色块。
- 保持主图纸区域与预览图完全解耦。

---

## 推荐组件拆分

### 页面层
`FocusModePage`

职责：
- 获取数据。
- 计算当前色块。
- 决定当前主图纸渲染参数。
- 维持预览图与主图纸的状态同步。

### 主图纸组件
例如：`FocusPatternZoomView`

职责：
- 接收窗口参数和色块数据。
- 执行局部放大 canvas 绘制。
- 绘制标尺与高亮。

### 预览图组件
例如：`FocusPatternOverview`

职责：
- 保持现有整图 canvas 方案。
- 不引入局部放大逻辑。
- 继续显示当前颜色在全图中的位置。

---

## 页面交互逻辑建议

1. 用户进入专注模式页面。
2. 页面根据当前 `projectId` 读取图纸状态。
3. 主图纸区域渲染当前色块局部放大图。
4. 预览图区域继续显示整图位置。
5. 当用户切换当前颜色时：
   - 主图纸局部窗口重新计算。
   - 标尺位置重新判断。
   - 主图纸重新绘制。
   - 预览图保持现有整图风格不变。

---

## 实施顺序建议

### 第一步
新增主图纸局部放大渲染器。

### 第二步
在专注模式页面中接入局部窗口计算逻辑。

### 第三步
根据色块中心判断四象限，配置标尺位置。

### 第四步
联调主图纸与预览图，确认预览图外观完全不变。

### 第五步
根据实际效果微调：
- `N` 的最小值和 padding。
- 标尺数字密度。
- 当前色块高亮强度。

---

## 验收标准

1. 预览图显示效果保持不变。
2. 主图纸区域不再展示整图，而是展示当前色块的局部放大图。
3. 局部窗口大小 `N` 自动计算，且不小于 `10`。
4. 主图纸的标尺位置能够根据当前色块所在象限正确切换。
5. 主图纸能够突出当前正在拼的颜色，帮助用户快速定位。
6. 预览图与主图纸渲染逻辑分离，互不影响。

---

## 当前实现核对与遗漏项

截至当前实现，专注模式主图纸局部放大能力已经完成以下内容：

1. **未选择色号时的主图纸整图显示**
   - 未选择色号时，主图纸区域会使用 `drawPatternPreview` 显示完整图纸。
   - 选择色号后，主图纸切换为 `drawPatternFocusZoomView` 局部放大模式。

2. **局部放大模式基础能力**
   - 当前色块默认显示在固定 `10 × 10` 主图纸窗口内。
   - 左手模式默认贴近当前色块右上角。
   - 右手模式默认贴近当前色块左上角。
   - 标尺位置按操作习惯显示：左手为上侧 + 右侧，右手为上侧 + 左侧。
   - 标尺厚度当前为 `18px`。

3. **受限滑动能力**
   - 当当前色块跨度超过 `10 × 10` 时，滑动范围会根据当前色块跨度限制。
   - 例如 `12 × 15` 色块，横向最多移动 `2` 格，纵向最多移动 `5` 格。
   - 不再允许主图纸窗口在整张图纸范围内无限拖动。

4. **工具栏横纵移动能力**
   - 原 `+` / `-` 按钮已替换为“横”“纵”两个移动按钮。
   - 当当前色块横向跨度超过 `10` 时，“横”按钮可用。
   - 当当前色块纵向跨度超过 `10` 时，“纵”按钮可用。
   - 每次点击按当前窗口大小 `10` 格进行分页式移动，并受当前色块滑动边界限制。

5. **大色块分页分割线**
   - 当前色块横向跨度大于 `10` 时，在第 `10` 格和第 `11` 格之间绘制竖向分页分割线。
   - 当前色块纵向跨度大于 `10` 时，在第 `10` 格和第 `11` 格之间绘制横向分页分割线。
   - 分割线只在对应分页边界进入当前 `10 × 10` 窗口时显示。

6. **预览图小地图定位框**
   - 预览图上已增加 `10 × 10` 定位框。
   - 定位框表示当前主图纸正在显示/编辑的位置。
   - 主图纸拖动或点击“横”“纵”移动后，定位框会同步移动。

7. **非当前色块弱化显示**
   - 局部放大模式中，当前正在拼的色块保持原色并高亮。
   - 其他非当前色块显示为灰白相间的棋盘格。
   - 透明格和外部格同样显示为灰白棋盘格。

### 当前遗漏或待补充事项

1. **文档中的 `N` 自适应计算尚未按原始方案实现**
   - 原方案提出 `N = Math.max(10, max(spanX, spanY) + padding * 2)`。
   - 当前实际实现中，主图纸可视窗口固定为 `10 × 10`。
   - 大色块通过受限滑动和横纵分页移动查看，不会把窗口自动扩展为 `12 × 15` 或更大。
   - 因此需要在方案中明确：当前产品决策已从“自适应扩大窗口”调整为“固定 `10 × 10` 窗口 + 受限滑动”。

2. **标尺数字密度仍未优化**
   - 当前标尺基本每格都会显示数字。
   - 虽然固定 `10 × 10` 时可读性尚可，但如果未来重新支持更大 `N`，需要增加标尺数字密度控制。

3. **主图纸组件拆分尚未完成**
   - 文档建议拆分 `FocusPatternZoomView` 和 `FocusPatternOverview`。
   - 当前实现仍主要集中在 `FocusModePage` 中，通过 canvas ref 和 effect 调用渲染函数。
   - 后续如继续扩展交互，建议拆分组件以降低页面复杂度。

4. **拼豆方案模式尚未真正影响主图纸排序和导航**
   - 设置面板中已有“智能模式、色块优先、边缘优先、区域优先、逐行/逐列模式”等选项。
   - 当前局部放大和块导航主要受色号、连通块、左右手操作习惯影响。
   - `patternMode` 暂未真正接入当前色块排序、色块拆分或导航策略。

5. **局部放大模式下非当前色块上下文被弱化为棋盘格**
   - 当前实现为了突出当前色块，已将其他色块显示为灰白棋盘格。
   - 这与早期“显示当前色块周边上下文”的描述存在差异。
   - 如果后续仍需要观察周边图案颜色关系，需要增加一个显示开关或弱化透明度方案。

6. **完成状态的持久化仍需确认**
   - 当前已完成格子和已完成颜色主要在页面 state 中维护。
   - 若需要退出专注模式后继续恢复进度，需要确认是否已接入项目级持久化存储。

7. **预览图定位框交互仍是只读**
   - 目前定位框会随主图纸移动同步更新。
   - 但用户还不能直接拖动预览图定位框来反向移动主图纸。
   - 如果希望预览图成为完整“小地图”，后续可以增加定位框拖动/点击跳转能力。

8. **横纵分页移动按钮的状态提示仍可增强**
   - 当前按钮仅通过 disabled 状态表示是否可移动。
   - 后续可以显示当前分页位置，例如 `横 1/2`、`纵 1/2`，帮助用户判断大色块还有多少区域未查看。

---

## 风险与注意事项

1. **窗口边界裁剪问题**
   - 当当前色块靠近边缘时，需要处理窗口越界修正。

2. **标尺密度问题**
   - 若 `N` 较大，标尺数字过密会影响阅读性，需要控制显示频率。

3. **性能问题**
   - 主图纸重新绘制频率较高时，应尽量避免无意义的重复渲染。

4. **一致性问题**
   - 主图纸的局部放大逻辑必须与预览图解耦，避免修改一处影响另一处。

---

## 总结

本方案的核心是：

- **预览图继续承担整图定位职责**，并增加 `10 × 10` 小地图定位框。
- **主图纸在未选择色号时显示完整图纸**，选择色号后切换为局部放大视图。
- **主图纸固定显示 `10 × 10` 局部窗口**，围绕当前正在拼的色块进行聚焦展示。
- **大色块通过受限滑动、横纵分页移动和分页分割线查看超出部分**。
- **标尺位置根据左手/右手操作习惯动态决定**。
- **当前色块保持原色高亮，其他色块以灰白棋盘格弱化显示**。
- **通过局部放大 + 标尺 + 小地图定位框，提升专注模式下的拼豆定位效率**。
