# 专注模式主图纸局部放大设计方案

## 目标

在不改变现有预览图效果的前提下，重构专注模式页面中的“主图纸区域”展示方式，使其从“整图显示”变为“当前正在拼的颜色的局部放大视图”，帮助用户更快定位正在拼的区域。

## 设计原则

1. **预览图保持不变**
   - 继续使用现有整图 canvas 渲染逻辑。
   - 继续作为“全局定位参考图”使用。
   - 不改变其绘制规则、缩放策略和交互方式。

2. **主图纸改为局部放大视图**
   - 只聚焦当前正在拼的颜色块。
   - 通过局部窗口放大，让用户更容易观察当前色块周围的上下文。
   - 主图纸区域不再承担整图预览职责。

3. **数据层统一，渲染层分离**
   - 两个区域共享同一份图纸数据 `patternResult`。
   - 预览图和主图纸使用不同的渲染函数。
   - 避免修改主图纸逻辑时影响预览图表现。

---

## 页面职责拆分

### 1. 预览图

**职责**：显示整张图纸中当前颜色的大概位置。

**要求**：
- 保持现有显示效果不变。
- 继续使用当前 canvas 渲染逻辑。
- 继续显示整图、完成状态、颜色高亮等全局信息。

### 2. 主图纸区域

**职责**：显示当前正在拼的颜色的局部放大区域。

**要求**：
- 根据当前色块计算局部窗口。
- 使用 `N * N` 的大格子进行展示。
- `N` 最小为 `10`。
- 根据当前色块所在图纸区域决定标尺位置。
- 只强调当前颜色，方便用户快速定位。

---

## 总体实现思路

推荐采用 **“主图纸新增局部放大 canvas 渲染器”** 的方案。

即：

- 现有预览图继续使用 `drawPatternPreview`，不做改动。
- 新增一个主图纸局部放大渲染函数，例如：`drawPatternFocusZoomView`。
- 专注模式页面根据当前颜色块，计算局部窗口参数后调用该渲染函数。

这样可以确保：

- 预览图逻辑稳定。
- 主图纸视觉效果独立可控。
- 后续可继续扩展标尺、放大倍数、辅助线等能力。

---

## 数据来源

专注模式页面已经具备以下核心数据：

- `patternResult`：完整图纸数据
- `currentColorEntry`：当前正在拼的颜色
- `currentBlock`：当前颜色对应的色块/块级信息
- `activeBlockCellKeys`：当前激活格子集合
- `completedCellKeys`：已完成格子集合

其中，主图纸局部放大视图主要依赖：

- `patternResult`
- `currentBlock.cells`
- `activeBlockCellKeys`

## 主图纸局部放大视图设计

### 1. 局部窗口大小 `N`

主图纸展示区域将不再渲染整张图，而是渲染一个局部窗口，窗口大小为 `N * N`。

#### 计算规则

- 基于当前色块的范围计算窗口大小。
- `N` 必须至少为 `10`。
- 为了保证上下文可读性，窗口大小建议在当前色块尺寸基础上加入 padding。

#### 推荐计算方式

```ts
N = Math.max(10, max(spanX, spanY) + padding * 2)
```

其中：

- `spanX = maxX - minX + 1`
- `spanY = maxY - minY + 1`
- `padding` 建议为 `2 ~ 4`

#### 说明

- 如果色块很小，则窗口更偏向展示周边上下文。
- 如果色块较大，则窗口自动扩展，以完整容纳当前色块。
- 如果色块本身已经大于 10，则以色块尺寸为主。

---

### 2. 局部窗口定位

局部窗口需要围绕当前色块进行布局，确保当前色块尽量处于窗口中心附近。

#### 步骤

1. 计算当前色块边界：
   - `minX`
   - `maxX`
   - `minY`
   - `maxY`

2. 计算当前色块中心点：
   - `blockCenterX = (minX + maxX) / 2`
   - `blockCenterY = (minY + maxY) / 2`

3. 根据象限决定默认窗口锚点：
   - **左上象限**：优先展示色块左上角附近的 `10 × 10`
   - **右上象限**：优先展示色块右上角附近的 `10 × 10`
   - **左下象限**：优先展示色块左下角附近的 `10 × 10`
   - **右下象限**：优先展示色块右下角附近的 `10 × 10`

4. 当色块尺寸大于窗口时：
   - 默认只展示当前象限对应的 `10 × 10` 局部内容。
   - 后续可通过滑动/拖动继续查看未显示的其他格子。

#### 边界处理

- 如果窗口左侧超出图纸，窗口整体右移。
- 如果窗口右侧超出图纸，窗口整体左移。
- 如果窗口上侧超出图纸，窗口整体下移。
- 如果窗口下侧超出图纸，窗口整体上移。

这样可以保证：

- 局部窗口始终合法。
- 默认展示内容更贴近当前色块所在象限。
- 后续可扩展为拖动查看完整大色块。

---

## 标尺显示规则

标尺位置由当前色块所在的图纸区域决定。

### 四象限判断

以整张图纸的中心为分界：

- `centerX = pattern.width / 2`
- `centerY = pattern.height / 2`

以当前色块中心进行象限判断：

- **左上**：`blockCenterX <= centerX` 且 `blockCenterY <= centerY`
- **右上**：`blockCenterX > centerX` 且 `blockCenterY <= centerY`
- **左下**：`blockCenterX <= centerX` 且 `blockCenterY > centerY`
- **右下**：`blockCenterX > centerX` 且 `blockCenterY > centerY`

### 标尺位置规则

| 色块所在区域 | 横向标尺 | 纵向标尺 |
| --- | --- | --- |
| 左上 | 上侧 | 左侧 |
| 右上 | 上侧 | 右侧 |
| 左下 | 下侧 | 左侧 |
| 右下 | 下侧 | 右侧 |

### 标尺作用

标尺不仅是视觉装饰，还应提供明确的定位信息：

- 横向标尺：显示局部窗口内的列坐标。
- 纵向标尺：显示局部窗口内的行坐标。
- 支持每隔若干格显示数字，其余位置显示短刻度。

---

## 主图纸渲染内容

主图纸局部放大视图建议只突出以下内容：

1. **当前正在拼的颜色格子**
   - 作为当前视觉焦点。
   - 高亮边框或更强对比色处理。

2. **当前色块周边上下文**
   - 让用户知道当前颜色块和周围图案的位置关系。
   - 保持与背景格子的区分。

3. **标尺**
   - 显示在上/下/左/右对应边。
   - 帮助用户进行精确定位。

4. **必要的格线**
   - 维持大格子结构感。
   - 让局部区域更像“放大后的图纸”。

### 不建议保留的内容

为了避免干扰定位，主图纸局部视图中不建议强化以下内容：

- 整图缩略信息。
- 与当前颜色无关的过多装饰。
- 与预览图重复的全局表达。

---

## 渲染层设计建议

### 1. 预览图渲染器保持原样

继续使用现有的 `drawPatternPreview` 逻辑。

它负责：

- 整图展示。
- 当前颜色高亮。
- 已完成格子覆盖。
- 分割线显示。

### 2. 新增主图纸局部放大渲染器

建议新增一个独立函数，例如：

```ts
drawPatternFocusZoomView(params)
```

建议入参包含：

- `canvas`
- `pattern`
- `currentBlockCells`
- `windowBounds`
- `rulerPosition`
- `activeColorKey`
- `completedCellKeys`

#### 主要职责

- 计算局部格子绘制位置。
- 绘制 `N * N` 放大网格。
- 绘制标尺。
- 高亮当前色块。
- 保持主图纸区域与预览图完全解耦。

---

## 推荐组件拆分

### 页面层
`FocusModePage`

职责：
- 获取数据。
- 计算当前色块。
- 决定当前主图纸渲染参数。
- 维持预览图与主图纸的状态同步。

### 主图纸组件
例如：`FocusPatternZoomView`

职责：
- 接收窗口参数和色块数据。
- 执行局部放大 canvas 绘制。
- 绘制标尺与高亮。

### 预览图组件
例如：`FocusPatternOverview`

职责：
- 保持现有整图 canvas 方案。
- 不引入局部放大逻辑。
- 继续显示当前颜色在全图中的位置。

---

## 页面交互逻辑建议

1. 用户进入专注模式页面。
2. 页面根据当前 `projectId` 读取图纸状态。
3. 主图纸区域渲染当前色块局部放大图。
4. 预览图区域继续显示整图位置。
5. 当用户切换当前颜色时：
   - 主图纸局部窗口重新计算。
   - 标尺位置重新判断。
   - 主图纸重新绘制。
   - 预览图保持现有整图风格不变。

---

## 实施顺序建议

### 第一步
新增主图纸局部放大渲染器。

### 第二步
在专注模式页面中接入局部窗口计算逻辑。

### 第三步
根据色块中心判断四象限，配置标尺位置。

### 第四步
联调主图纸与预览图，确认预览图外观完全不变。

### 第五步
根据实际效果微调：
- `N` 的最小值和 padding。
- 标尺数字密度。
- 当前色块高亮强度。

---

## 验收标准

1. 预览图显示效果保持不变。
2. 主图纸区域不再展示整图，而是展示当前色块的局部放大图。
3. 局部窗口大小 `N` 自动计算，且不小于 `10`。
4. 主图纸的标尺位置能够根据当前色块所在象限正确切换。
5. 主图纸能够突出当前正在拼的颜色，帮助用户快速定位。
6. 预览图与主图纸渲染逻辑分离，互不影响。

---

## 风险与注意事项

1. **窗口边界裁剪问题**
   - 当当前色块靠近边缘时，需要处理窗口越界修正。

2. **标尺密度问题**
   - 若 `N` 较大，标尺数字过密会影响阅读性，需要控制显示频率。

3. **性能问题**
   - 主图纸重新绘制频率较高时，应尽量避免无意义的重复渲染。

4. **一致性问题**
   - 主图纸的局部放大逻辑必须与预览图解耦，避免修改一处影响另一处。

---

## 总结

本方案的核心是：

- **预览图继续承担整图定位职责**，保持现有 canvas 渲染逻辑不变。
- **主图纸改为局部放大视图**，围绕当前正在拼的颜色进行聚焦展示。
- **窗口大小 `N` 自适应计算，最小为 10**。
- **标尺位置根据色块中心所在象限动态决定**。
- **通过局部放大 + 标尺辅助，提升专注模式下的拼豆定位效率**。
